Opening a File and Reading its Contents

def getInventoryFile():
                                                       
# Get inventory file and read into a dictionary KEY: product code, VALUE: number of items left on shelf.

    inventoryFile = 'inventory.txt'
    
    inventoryDict = {}
    
    with open(inventoryFile,'r') as f:
        for line in f:          
            line = line.split('#') 
            inventoryDict[line[0]]=(line[1].strip()) 
        return inventoryDict

A List Holding Functions and Arguments

# bundling functions into a list or dictionary and then calling the functions from the list

# set of functions

def func1(text):
    print(text)

def func2(text):
    print(text)

def func3(text):
    print(text)

def newFunc():
    print('newfunc')

# List of lists holding pointers to each function and the argument. [[functionHere,argument here],[...]]

list1=[[func1,'argument for func1'],[func2,'argument for func2'],[func3,'argument for func3',]] # Note: the function does NOT take its parantheses.


# access functions through the list of lists [[functionHere,argumentHere],[functionHere,argumentHere]]

print('\nlist calls')
for list in list1: # access the list in the list
    list[0](list[1]) # access each item in the internal list: run the func (located at list[0]) with its arguments

# Dictionary version: note you can not have duplicate keys in a dict which will be a problem when sequencing function calls

funcDict={func1: "func1",func2: "func2",func3: "func3"}

print(funcDict)

print('dictionary calls here:')
for k,v in funcDict.items():
    k(v)

# Create object
root = Tk()

# Adjust size
root.geometry("400x400")

Python: Pickling

Pickle allows you to save and load Python data structures and code. 

import Pickle

def saveFile(dataToSave, filename):
    with open(filename, "wb") as fp:  # Pickling
        pickle.dump(dataToSave, fp)
        fp.close()


def loadFile(filename):
    with open(filename, "rb") as fp:  # Unpickling
        return pickle.load(fp)

    pixelColour = screenshot.getpixel(mousePos)

    time.sleep(2)

        self.master.geometry("+150+500")  # position of the window in the screen (200x300) ("-3300+500")

        # if not set, the frames will fill the master window

        screenHeight = self.master.winfo_screenheight()

        self.master.attributes("-topmost", True)

        # Instantiate frames

        self.frame1 = Frame(self.master, bd=5, padx=5, bg='#2a2b2b')  # Side Column
        self.frame2 = Frame(self.master, bd=5, padx=5, bg='#7E050C')  # Main frame

Tkinter: Using Lambdas in Button Commands

In Tkinter the buttons will normally need to be connected with a function. These functions get connected by the "command" option inside the Button class. However if you simply type in the function with parantheses (and possibly also some arguments) then the function will get called at the moment the program creates the button. This is very likely not what you want. Rather we want buttons to call their function only when the user clicks the button, not at any othertime. In order to make this the case, use lambda before the function as shown below. The second example below shows what to do with functions which do not take any arguments. They need to have their parantheses. 

 

# First Example (connecting a function which has parameters)
button1=Button(self.frame1, text=description[0], width=12,command=lambda:self.schedule('This button works.'))
button1.pack() # also make sure to put the pack() as a separate call not connected with the Button instantiation (not all on one line).

# Second Example (connecting functions which do not have parameters but still keeping the paranthesis)
self.createButton = Button(self.frame1, text="Create", width=12, command=lambda:self.createAutomation()) # Functions which do not take arguments but still have parantheses

# Final Example (connecting functions which do not have parameters and omitting parantheses as well as the lambda)
self.createButton = Button(self.frame1, text="Create", width=12, command=self.createAutomation) # Functions which do not take arguments and omitting the parantheses

Python: Making Imports

from x import *

The above syntax imports all items from a module but without giving a prefixed name to those items; this brings all items from the external module directly into the namespace so no prefix is needed; it does make for less explicit coding however and can lead to collisions in the namespace.

import x as tk

This gives the imported module its own prefixed name, in the above case, tk; when you need to use a function from that module you will write: tk.nameOfFunction

from x import function1, function2, function3...

The above syntax will import only the functions needed rather than filling the name space with all functions from the external module.

import x

The above method imports all functions from the module x and also requires a prefix of the same name of the module. In the above case like this: x.myFunction
